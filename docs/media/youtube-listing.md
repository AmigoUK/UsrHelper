## Description

```
Report a bug in two clicks. Take a screenshot or record your screen, mark what is wrong, and get a complete report — no follow-up questions needed.

It captures what a developer always has to ask for anyway: the page address, the exact browser and system, your last clicks, and the errors the page printed.

UsrHelper is a free, open source Chrome extension for testers, support staff and end users. No account, no sign-in, and nothing is ever uploaded: every file is saved to your own Downloads folder.

⬇️ Install from the Chrome Web Store:
https://chromewebstore.google.com/detail/usrhelper/hmmlhdogplekofonkkmhacfolcfgejic

📖 Guide with pictures (English and Polish):
https://amigouk.github.io/UsrHelper/

💻 Source code (MIT licence):
https://github.com/AmigoUK/UsrHelper

💡 Request a feature or report a problem:
https://attv.uk/projects/usrhelper.html


WHAT IT DOES

Screenshots
• Capture the visible area, the whole page (it scrolls and stitches automatically), or a region you drag
• Annotate: marker, rectangle, ellipse, arrow, text, numbered step markers, crop, undo and redo
• Yellow sticky notes for a comment of a sentence or two — numbered on the image, and repeated as text in the report file so a developer can copy them instead of retyping
• Anonymisation brush: paint a pixel mosaic over customer names, invoices or tokens. It is baked into the exported image, so the hidden part is gone for good
• Click path: the last ten places you clicked before capturing, stamped on the image as numbered markers

Screen recording
• Record a tab, a window, or the whole screen, with microphone narration
• Microphone level meter, 3-2-1 countdown, pause and resume, elapsed timer
• Optional camera bubble and a clock burned into the video
• Your clicks appear as ripples; keyboard shortcuts appear as captions, while ordinary typing is never shown — a password typed during a recording does not end up on screen
• Long recordings split into standalone clips automatically, so a session never becomes one giant attachment

The report
• Files saved to a folder of your choice inside Downloads, with the date and time in the name
• A companion JSON file with your description, your sticky notes, the page address and title, your click path, JavaScript errors from the page, and the environment: operating system and version, processor type, exact browser version and screen size
• One click opens a ready-made email with recipients, subject and description already filled in
• Project profiles you can export to a file and hand to your testers, so nobody has to type the right address by hand


WHY IT IS DIFFERENT

• Nothing is uploaded. Screens from software under test often show customer data; here they never leave your machine.
• No account and no team to join. Install it and use it.
• It works offline. There is no service that can be slow, down, or discontinued.
• Free and open source under the MIT licence — anyone can read the code and check what it really does.
• The report describes your real computer. Browsers hide the true system version and report every Apple Silicon Mac as an Intel Mac; UsrHelper reads the accurate values instead.
• Recording keeps its frames when you switch tabs, because the work happens in a separate worker rather than in the page.
```
